# 🐙 Git — Шпаргалка для новичка

---

## 🔧 Установка и настройка (один раз)

### Установить Git
Скачай на **git-scm.com** и установи (всё по умолчанию).

Проверь что установился:
```bash
git --version
```
Должно появиться что-то вроде: `git version 2.43.0`

### Представиться Git (один раз на компьютере)
```bash
git config --global user.name "Твоё Имя"
git config --global user.email "твой@email.com"
```

## 📁 Создать новый проект

### 1. Создай папку и зайди в неё
```bash
mkdir C:\Projects\мой-проект
cd C:\Projects\мой-проект
```
Или просто создай папку в Проводнике, потом открой PowerShell в ней.

### 2. Инициализируй Git в папке
```bash
git init
```
Git создаст скрытую папку `.git` — это и есть вся история проекта.

### 3. Подключи к репозиторию на GitHub
```bash
git remote add origin https://github.com/ТвоёИмя/ТвоёРепо.git
```
Ссылку берёшь со страницы репо → кнопка **Code → HTTPS**.

## 📤 Загрузить файлы на GitHub (первый раз)

```bash
git add .
git commit -m "Первый коммит"
git branch -M main
git push -u origin main
```

**Что это значит по шагам:**

| Команда | Что делает |
|--------|-----------|
| `git add .` | Говорит Git: "запомни ВСЕ изменения в папке" |
| `git commit -m "текст"` | Сохраняет снимок с описанием что изменил |
| `git branch -M main` | Называет главную ветку `main` |
| `git push -u origin main` | Загружает на GitHub |

> ⚠️ При первом push Git попросит логин и пароль.  
> Вместо пароля вставляй **токен** (`ghp_...`) с github.com/settings/tokens

## 🔄 Обновить проект (каждый раз)

Изменил файл → сохранил → три команды:

```bash
git add .
git commit -m "Что изменил"
git push
```

Всё. GitHub обновится автоматически за ~30 секунд.

---

## 📥 Скачать проект с GitHub на компьютер

```bash
git clone https://github.com/ТвоёИмя/ТвоёРепо.git
```

Создаст папку с проектом прямо здесь.

## 🔍 Полезные команды

### Посмотреть что изменилось
```bash
git status
```
Покажет какие файлы изменены, добавлены или удалены.

### Посмотреть историю коммитов
```bash
git log --oneline
```
Короткий красивый список всех сохранений.

### Посмотреть конкретные изменения в файле
```bash
git diff
```
Покажет строки которые добавил (+) или удалил (-).

## ⏪ Откат — если что-то пошло не так

### Отменить изменения в файле (вернуть к последнему коммиту)
```bash
git checkout -- имяфайла.html
```

### Отменить последний коммит (файлы останутся)
```bash
git reset --soft HEAD~1
```

### Принудительно перезаписать GitHub своей версией
```bash
git push --force
```
> ⚠️ Осторожно — удалит всё что было на GitHub!

## 🌿 Ветки (для более сложных проектов)

Ветка = отдельная копия проекта для экспериментов.

```bash
git branch новая-идея        # создать ветку
git checkout новая-идея      # переключиться на неё
git checkout main            # вернуться на главную
git merge новая-идея         # влить ветку в main
git branch -d новая-идея     # удалить ветку
```

## 🔑 Токен GitHub

Токен нужен вместо пароля при push.

1. Зайди на **github.com/settings/tokens**
2. **Generate new token (classic)**
3. Поставь галочку ✅ **repo**
4. Нажми **Generate token**
5. Скопируй сразу — показывается только один раз!

Выглядит так: `ghp_xxxxxxxxxxxxxxxxxxxxxx`

## 🚀 Шпаргалка на стену

```
git status              → что изменилось?
git add .               → добавить все изменения
git commit -m "текст"   → сохранить с описанием
git push                → загрузить на GitHub
git pull                → скачать с GitHub
git log --oneline       → история коммитов
git clone [ссылка]      → скачать проект
```

## 💡 Типичные ошибки и решения

**`error: src refspec main does not match any`**
→ Папка пустая, добавь файл, потом `git add .` и `git commit`

**`rejected — fetch first`**
→ На GitHub есть изменения которых нет у тебя:
```bash
git pull origin main --allow-unrelated-histories
git push
```

**`CONFLICT`**
→ Конфликт файлов. Оставить свою версию:
```bash
git checkout --ours имяфайла.html
git add .
git commit -m "resolved"
git push
```

**Забыл закрыть кавычку в commit**
→ Нажми `Ctrl+C` и введи заново с закрытой кавычкой:
```bash
git commit -m "описание"
```
